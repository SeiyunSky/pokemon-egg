# 玩法设计文档 v1.0

## 核心体验定位

**宠物是个有收藏癖的小家伙。**  
它会发现东西、攒东西、把东西带回家摆好，然后叫朋友来看。  
玩家的角色不只是照顾者，也是这个小世界的见证者和布置者。

---

## 核心循环

```
日常照顾（喂食/玩耍/抚摸/休息）
    ↓ 触发
探索 / 日常事件 → 获得收藏品
    ↓ 玩家决策
摆进小屋 或 归入收藏册
    ↓ 积累效应
解锁新话术 / 新互动动作 / 好友来访资格
    ↓ 好友来访
朋友聚会（多角色 AI 对话场景，基于收藏品和状态生成）
    ↓ 产出
AI 日记（以当天收藏、聚会、互动为素材）
    ↓ 可分享
朋友看到你的小屋 / 聚会记录 / 收藏册
```

## 一、收藏品系统

### 1.1 收藏品分类

| 类型 | 说明 | 举例 |
|------|------|------|
| `object` | 有形的实物 | 一颗奇怪的石头、旧照片、别人的帽子、泥巴球、布料碎片 |
| `memory_fragment` | 某个瞬间被记录下来的片段 | "那天主人迟到了两小时"、"第一次下雨时我们都没有伞" |
| `skill_quirk` | 宠物学会的技能或癖好 | 学会了侧翻、喜欢闻书的味道、会假装睡着 |
| `friend_gift` | 好友专程带来的东西 | 乌波的泥巴球、谜拟Q的布料碎片、差不多娃娃咬过的玩具 |
| `seasonal` | 特殊日期/节日限定 | 生日蜡烛、节日小挂件、限定徽章 |

### 1.2 收藏品数据结构

```js
{
  id: "col_xxx",
  type: "object" | "memory_fragment" | "skill_quirk" | "friend_gift" | "seasonal",
  
  // 外观（展示用）
  name: "一顶小小的帽子",          // AI 生成 or 模板填充
  description: "不知道是谁的，...", // AI 生成，每件独特
  emoji: "🎩",                      // 固定表情符号，用于收藏册格子展示
  cssClass: "item-hat",             // 小屋里的像素块样式
  rarity: "common" | "rare" | "unique",
  
  // 来源
  source: "event" | "friend_visit" | "exploration" | "seasonal",
  sourceDetail: "rainy_day_event",  // 具体来源 id
  acquiredAt: 1746787200000,
  
  // 效果（解锁内容）
  unlocks: [
    { type: "dialogue_line", id: "hat_talk_01" },   // 宠物会提到这顶帽子
    { type: "interaction",   id: "wear_hat" },       // 解锁"给宠物戴帽子"互动
    { type: "visit_trigger", friendId: "mimikyu" },  // 达到条件谜拟Q会来
  ],
  
  // 小屋摆放状态
  placement: {
    inRoom: true,
    position: { slot: 3 },         // 小屋的第几个槽位
    inAlbum: false,
  },
  
  // AI 生成元数据
  isAiGenerated: true,
  promptTemplate: "object_found_v1",
}
```

### 1.3 收藏品来源

#### 日常事件掉落
每次事件有一定概率掉落收藏品，掉落物由 AI 根据事件内容生成（模板约束 + AI 填充）。

**AI 生成模板示例：**
```
事件：下雨天，宠物趴在窗边发呆
模板：object_found_outdoor
AI 生成：{
  name: "一片很大的叶子",
  description: "雨停之后它叼进来的，叶子上还有雨水的痕迹，
                干了之后留下了很漂亮的脉络。"
}
```

#### 好友来访带来
每个好友有固定的"带礼物概率"和专属礼物池，偶尔带来独特物品。
- 乌波：泥巴相关、地面系物品
- 谜拟Q：布料碎片、伪装道具
- 差不多娃娃：咬过的东西、粉色物品

#### 宠物主动探索
消耗精力，派宠物出去探索（1-3小时），回来带东西。
探索方向影响掉落类型：
- 森林 → `object`（自然类）
- 城镇 → `object`（人工类）+ `memory_fragment`
- 海边 → `object`（水系）+ 偶发好友线索

#### 节日/特殊日期
宠物生日、节假日触发限定收藏品，每年限一次，不可重复获得。

## 二、小屋系统

### 2.1 视觉形式
横板剖视小屋，像素风，暖色调。屋内有若干**固定槽位**（书架/地板/桌面/墙壁），收藏品占据不同槽位，有对应的像素块外观。

```
┌─────────────────────────────────────────┐
│  [墙壁槽×3]   [窗户]   [墙壁槽×2]      │
│                                         │
│  [书架槽×4]                             │
│                                         │
│  [地板槽×6]        [宠物在这里]         │
└─────────────────────────────────────────┘
```

槽位有类型限制（`wall` / `shelf` / `floor`），部分收藏品只能放特定位置。

### 2.2 小屋交互
- 点击槽位中的收藏品 → 弹出收藏品详情卡（名称 + AI 生成描述 + 获取故事）
- 点击宠物 → 宠物对话，若当前小屋有特定收藏品，宠物会主动提及
- 长按收藏品 → 拖拽调整位置（同类型槽位间交换）

### 2.3 小屋等级（隐性）
收藏品数量达到阈值时，小屋自动解锁新区域或新槽位，不需要玩家操作，属于自然成长。

## 三、收藏册系统

### 3.1 形式
翻页式收藏册，每页 3×4 格，格子里显示收藏品 emoji + 名称。
未获得的格子显示问号（部分连轮廓也不显示，制造惊喜感）。

### 3.2 分页
- 第一页：日常物件
- 第二页：记忆碎片
- 第三页：技能与癖好
- 第四页：好友礼物
- 第五页：节日限定

### 3.3 完成度奖励
某一页集满时触发特殊事件或解锁新好友来访。

## 四、探索系统

### 4.1 流程
1. 玩家选择探索方向（森林 / 城镇 / 海边）
2. 消耗精力（-20 to -30），设定等待时间（1-3小时，按方向而定）
3. 等待期间宠物不可互动（在外面）
4. 回来后触发"探索报告"事件，AI 生成探索经历描述 + 掉落物

### 4.2 探索报告（AI 生成）
**传入数据：**
- 宠物性格、种类
- 探索方向
- 探索时长
- 掉落物（已由系统决定，AI 负责编故事）

**输出格式：**
```json
{
  "story": "它去了城镇的边缘，在一棵歪脖子树下坐了很久...",
  "item_story": "带回来的那顶帽子是在垃圾桶旁边发现的，
                 帽檐上有一个很小的名字，但已经看不清了。"
}
```

## 五、朋友聚会系统

### 5.1 来访触发条件
每个好友有独立的来访逻辑：

| 好友 | 解锁条件 | 来访频率 | 特殊触发 |
|------|---------|---------|---------|
| 乌波 | 亲密度 ≥ 20 | 每24h一次 | 宠物有水系收藏品时必来 |
| 谜拟Q | 亲密度 ≥ 30 + 拥有布料类收藏品 | 每48h一次 | 宠物心情 < 40 时触发安慰来访 |
| 差不多娃娃 | 亲密度 ≥ 25 | 每36h一次 | 宠物进化时庆祝来访 |

### 5.2 聚会场景（AI 对话）
来访时进入聚会场景，小屋背景，好友和宠物都在场。

**AI 生成多角色对话，参数：**
- 主角宠物：性格 + 当前状态 + 近期收藏品
- 来访好友：固定性格模板
- 场景触发原因（日常/安慰/庆祝/好奇某件收藏品）
- 小屋当前陈设（影响对话话题）

**输出：** 3-6 轮对话，每轮有角色标记，结束时可能掉落收藏品或产生记忆碎片。

### 5.3 玩家参与
聚会结束时，玩家可以选择：
- "记录这次聚会" → 生成聚会日记条目，存入收藏册
- "送好友一件东西" → 消耗一件收藏品，提升好友亲密度

## 六、每日任务 / 回归动机

### 6.1 每日签到奖励
简单登录即可领取，内容由 AI 生成（基于当日日期/季节/宠物状态），
不是固定的"金币×10"，而是有趣的小东西：
- "今天捡到一片写着字的树叶"
- "收到了一张不知道谁寄来的明信片"

**AI 生成模板：** `daily_login_{season}_{mood_level}`，产出一条收藏品描述。

### 6.2 宠物主动联系
状态危急时（hunger < 20 / mood < 20），宠物会"发消息"（浏览器通知 or 页面内提示），
带有 AI 生成的宠物语气文案。

### 6.3 好友来访提醒
好友来访时的通知，带上好友名字和本次带来的东西预告（制造期待感）。

### 6.4 亲密度里程碑解锁
| 亲密度 | 解锁内容 |
|--------|---------|
| 10 | 宠物开始有口头禅 |
| 25 | 解锁"探索"功能 |
| 40 | 解锁第一个好友来访 |
| 60 | 宠物对话时偶尔说人话 |
| 80 | 解锁小屋第二区域 |
| 100 | 解锁"共同记忆"收藏品（独特） |

## 七、信件系统

**参考：动物森友会信件机制**

### 7.1 信件来源

| 来源 | 触发条件 | 是否 AI 生成 |
|------|---------|-------------|
| 好友来信 | 每周随机 / 进化后 / 节日 | 是（依据 letters.js 模板） |
| 系统信件 | 第一次孵化 / 进化 / 生日 | 否（固定模板，带变量替换） |
| 玩家写信给宠物 | 玩家主动触发 | 宠物用 AI 回信 |
| 玩家互送信件 | Phase 后期，社交功能 | 玩家手写 |

### 7.2 信件结构
- 信件可附带收藏品或道具
- 读完后收入收藏册"信件"页
- 宠物回信有 2-8 小时延迟（模拟真实感）

### 7.3 好友推荐商品
好友来信时可附带商品推荐链接，享折扣价，不占商店槽位。

## 八、随机商店系统

### 8.1 刷新机制
- 每天固定时间刷新（默认早 6 点）
- 每次从三个商品池中随机抽取 6 件上架
- 同一件商品不会连续两次出现
- 货币来源：日常互动 / 完成事件 / 连续登录奖励

### 8.2 商品池分层

| 池 | 出现概率 | 商品类型 | 解锁条件 |
|---|---------|---------|---------|
| 常规池 | 高 | 基础道具（浆果/玩具） | 无 |
| 稀有池 | 低 | 特效道具 | 亲密度 / 进化阶段 |
| 限定池 | 季节/节日 | 限时道具 | 特定时间段 |

### 8.3 配置文件
`src/data/content/shop.js` — 所有商品定义、价格、出现权重。

## 九、社交 / 分享

**定位：分享服务于收藏，不是独立玩法。**

| 功能 | 说明 |
|------|------|
| 分享小屋截图 | 生成带宠物名字的定制图，可发给朋友 |
| 分享聚会记录 | 把 AI 生成的聚会对话做成"故事卡"分享 |
| 分享日记 | 把某篇日记做成图文卡片 |
| 查看朋友的收藏 | （Phase 后期）看朋友的收藏册，互送一件收藏品 |

## 十、AI 调用汇总

| 功能 | 触发时机 | 传入 | 输出 |
|------|---------|------|------|
| 收藏品生成 | 事件掉落 / 探索回来 / 签到 | 来源事件 + 宠物性格 + 模板类型 | 收藏品名称 + 描述 |
| 探索报告 | 探索结束 | 探索方向 + 宠物性格 + 掉落物 | 经历故事 + 物品来源故事 |
| 聚会对话 | 好友来访 | 双方性格 + 收藏品 + 触发原因 | 3-6 轮多角色对话 |
| 签到奖励文案 | 每日登录 | 日期 + 季节 + 宠物状态 | 一条有趣的收藏品描述 |
| 宠物催促文案 | 状态危急 | 宠物性格 + 危急状态类型 | 1-2 句宠物语气通知文案 |
| 好友来信 | 每周随机 / 特定事件 | 好友性格 + 触发原因 + 宠物近况 | 信件正文 |
| 宠物回信 | 玩家写信后 2-8 小时 | 宠物性格 + 玩家信件内容 + RAG记忆 | 回信正文 |

## 十一、数据模型增量

### 新增模型
- `Collectible`：单件收藏品（含 placement 状态）
- `Room`：小屋状态（槽位映射）
- `Album`：收藏册状态（完成度）
- `Exploration`：探索任务（进行中 / 完成）
- `GatheringRecord`：聚会记录
- `Letter`：信件（已读/未读、附带物品、回信状态）
- `ShopRefresh`：商店当前上架记录（含刷新时间戳）

### 现有模型调整
- `Pet` 新增：`explorationStatus`（当前是否在外探索）
- `Friend` 新增：`visitConditions`、`giftPool`
- `EventLog` 新增：`collectibleDrop`

## 十二、可编辑内容文件索引

所有游戏内容（文案/模板/数值）集中在 `src/data/content/`，与代码完全解耦：

| 文件 | 编辑内容 |
|------|---------|
| `collectible-templates.js` | 收藏品生成模板、emoji池、稀有度 |
| `exploration-templates.js` | 探索方向参数、场景氛围描述 |
| `gathering-scripts.js` | 聚会场景触发条件、话题种子 |
| `friend-profiles.js` | 好友性格、来访条件、礼物池 |
| `dialogue-lines.js` | 收藏品触发的话术、亲密度解锁台词 |
| `daily-rewards.js` | 签到奖励池、连续登录奖励 |
| `letters.js` | 好友来信模板、系统信件、宠物回信风格 |
| `shop.js` | 商品定义、价格、刷新配置 |
| `ai-prompt-templates.js` | 所有 AI 功能的 prompt 模板 |
